Where to Start #

Please use the following steps for the best user experience:

  1. Download LaTEX for Windows on: https://www.tug.org/texlive/acquire-netinstall.html (https://www.tug.org/texlive/acquire-netinstall.html).

  2. Download the Latex Cirriculum Vitae app on: https://github.com/saigkill/latex_curriculum_vitae-dotnet/releases (https://github.com/saigkill/latex_curriculum_vitae-dotnet/releases).

  3. Move the executable to a chosen location on your computer (for example, the desktop or Userdirectory). The app will be executed there, without any further installation.

  4. Launch the app.

User Settings #

When you launch latex_curriculum_vitae the first time, a User Settings window will open. Here you can fill in your personal information.

All settings in this window are mandatory.

If you want to shorten your links before they are saved, you can use BitLy. Get a token via https://bitly.is/accesstoken (https://bitly.is/accesstoken) and place it in the "BitLy Token" field.

The button saves your changes and closes the window. The button closes the window without applying changes to the config file. If you are click the button: you will be forwarded to this online documentation.

During the first run of latex_curriculum_vitae a collection of required files will be installed to: %appdata%/latex_curriculum_vitae. As an example, on my computer the full file path appears as: C:\Users\Sasch\Appdata\Roaming\latex_curriculum_vitae. Alternatively, if you type %appdata%/latex_curriculum_vitae in the Windows Explorer address field, Windows Explorer will direct you to the file path location.

The AppData directory holds all relevant data for building your job application.

After required files have been automatically installed and your user information has been updated, close the app and complete the following directory set-up before creating your first application.

Your AppData directory will look like this:

The Appendix directory contains 3 sub-directories:

  • Bibliography

  • Certificates

  • Certificates of Employment

The Bibliography directory only holds a file called bibliography.bib. This is a standard bibfile from LaTEX, currently containing my publications. Feel free to replace these publications with your own.

Important

If you are not using this feature, comment out the sections \addbibresource{bibliography.bib} and: 

\section{Publikationen}
\printbibsection{article}{Artikel}
\printbibsection{book}{Bücher} 
\printbibsection{misc}{other publications}

of the curriculum_vitae.tex file, by placing a % symbol at the beginning of each specified line.

The Certificates directory is empty. In this directory, you can place copies of all your certificates (for example, certificates from further education or a completed course). The Certificates of Employment directory is reserved for that specific type of certificate, such as a letter of recommendation. To set-up the Certificates and Certificates of Employment directories, some additional features are important.

In most countries, you have to order your material anti-chronologic. That means the first certificate displayed is the oldest one, and the remainder are ordered from oldest to newest certificate. To create this structure, you need to add a prefix to each certificate file name. For example, if you have three certificates from jobs dated 2018, 2019, and 2020, then the certificate file prefixes would be named:

Example 2.1: Certificate File Naming Example #

  • 1_your_2018job.pdf

  • 2_your_2019job.pdf

  • 3_your_2020job.pdf

The next file is theoretically a 4_your_2021job.pdf. By preparing the file names this way, you can easily reverse the order in the app itself.

The Curriculum_Vitae directory contains a predefined LaTEX file called curriculum_vitae.tex, and a file named friggeri-cv.cls. As you might expect, these are the class files for producing the curriculum vitae.

The Letter of Application directory contains the letter_of_application.tex file, which is used to produce the application letter.

The Pictures directory contains some icons for building the CV, a signature.png, and a userpic.jpg. You can replace these files with your personal signature file and userpic by writing over the delivered one. Note: Keep the original file names.

There are two other files found in your AppData directory. By default, the JobApplications.csv is empty on the first run, and will be populated during the job application creation. You can open this CSV file using Excel or LibreOffice Calc.

Important

If you want to open and change the CSV file, make a copy first, and leave the original untouched.

The personal_data.tex contains your personal data. After you set your information in the User Settings Window, this file will be produced.

Important

This TEX file will be re-created after each iteration. If you need to change your information, use the User Settings Window.

Preparing the LaTEX Files #

As previously noted, the app directory contains two LaTEX files: letter_of_application.tex and curriculum_vitae.tex. Both files will be updated with your personal information. Note, it is possible to use your own TEX files, but they must be saved as letter_of_application.tex and curriculum_vitae.tex in the correct directories.

When run, the app produces two files for compiling the job application. The first, personal_data.tex, was discussed above. As an example, in my case, it looks like this: 

\def\firstname{Sascha}
\def\familyname{Manns}
\def\mystreet{Maifeldstraße 10}
\def\mycity{56727 Mayen}
\def\myphone{+49-000-0000000}
\def\myemail{Sascha.Manns@outlook.de}
\def\myblog{http://saschamanns.de}

If you add an \input{personal_data.tex} to your curriculum_vitae.tex, you can use the newly defined macros. For each place in the document where you write \firstname, LaTEX will set your first name.

If you prepare your own CV, and want to add an image reference or link with an image, you have to set the full path as: \href{https://www.facebook.com/sascha.manns}{sascha.manns}\includegraphics[width=0.4cm]{$HOME/AppData/Roaming/latex_curriculum_vitae/Pictures/facebook.png}.

The second file generated by the app is called application_details.tex. It will look something like this:

\def\jobtitle{Fachinformatiker Anwendungsentwicklung}
\def\company{XYZ GmbH}
\def\contact{Mr. Bean}
\def\street{Street 2}
\def\city{A City}
\def\salutation{Dear Mr. Bean,}
\def\subject{Application as Fachinformatiker Anwendungsentwicklung}
\def\addressstring{XYZ GmbH\\Mr. Bean \\Street 2\\56068 A City}

Because this file will only be created inside a temporary directory, you can use \input{application_details.tex} for the macros defined there.
